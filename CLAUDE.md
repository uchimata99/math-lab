# מעבדת המספרים — math-lab

אפליקציית לימוד לילדה בת 7: מתמטיקה כיתה ב' (מודל לומד + מסלול), קריאה בעברית, ומעבדת מספרים מתקדמת. קובץ יחיד: `index.html` (React + Babel-standalone, נטען מ-CDN). מתארח ב-GitHub Pages.

## עקרונות
- כל הקוד ב-`index.html`. אחרי שינוי במחוללי תרגילים — לאמת ב-Node (חילוץ הפונקציה והרצת בדיקות שכל תשובה עוברת את ה-`check` של עצמה).
- שומרים על מספרים/טווחים תקינים, RTL, וניקוד עברי.
- לא נוגעים במפתחות זמן-ריצה של כלי ההערות ב-localStorage (`livenotes_*` / `ln_*`).

## מנגנון ההערות החי (live-notes)
הערות מהאפליקציה נשלחות כ-GitHub Issues עם התווית `note`. כשהמשתמש אומר **"תריץ את ההערות"**:

1. `gh issue list --repo uchimata99/math-lab --label note --state open` (או דרך כלי ה-GitHub MCP).
2. לכל Issue: לקרוא את הגוף (כותרת + שורות `[מיקום]` ו-`• מה לתקן`).
3. ליישם כל הערה לפי המיקום שלה (שם המסך/הנושא/הרמה) ועקרונות הפרויקט.
4. לאמת (בדיקות Node + טעינת האפליקציה בדפדפן אם רלוונטי), `commit` + `push`.
5. לסגור את ה-Issue עם הערה "תוקן ✓".

מבנה גוף ה-Issue:
```
הערות — uchimata99/math-lab
========================

[מיקום ההערה]
• מה לתקן
```

**הפעלת הווידג'ט:** כניסה חד-פעמית לכתובת עם `?notes=1` (כיבוי: `?notes=0`). רדום למשתמשי קצה.
